# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities privately, **not** through public issues
or pull requests.

Use GitHub's private vulnerability reporting: open the repository's **Security**
tab and choose **Report a vulnerability**. This opens a private advisory visible
only to the maintainers.

Please include enough detail to reproduce the issue (affected version, a minimal
proof of concept, and the impact you observed). We will acknowledge your report
and keep you updated on the fix and disclosure timeline.

## Supported versions

Security fixes are made against the latest released version. Older versions are
not maintained; please upgrade before reporting an issue against them.
