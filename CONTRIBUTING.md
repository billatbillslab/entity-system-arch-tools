# Contributing

Thanks for your interest in contributing.

## Ground rules

- Open an issue to discuss substantial changes before sending a pull request, so
  we can agree on the approach.
- Keep pull requests focused: one logical change per PR, with a clear
  description of what it does and why.
- Match the style and conventions of the surrounding code.

## Developer Certificate of Origin (sign-off)

This project uses the [Developer Certificate of Origin](https://developercertificate.org/)
(DCO) instead of a CLA. By signing off on a commit you certify that you wrote the
patch or otherwise have the right to submit it under the project's
[LICENSE](LICENSE).

Sign off every commit by adding a `Signed-off-by` line — `git commit -s` does this
for you:

```
Signed-off-by: Your Name <you@example.com>
```

Use your real name and a reachable email. Pull requests whose commits are not
signed off cannot be merged.

## Reporting bugs

Open an issue with a minimal reproduction, the version you are on, and what you
expected versus what happened. For security issues, follow
[SECURITY.md](SECURITY.md) instead — do not open a public issue.
